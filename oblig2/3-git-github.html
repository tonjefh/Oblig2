Git/GitHub

Det er i dag blitt viktigere å hente informasjon om filer og prosjekter rakt og effektivt. Ved å bruke tjenesten GitHub kan man benytte seg verktøyet git. Ved å ha prosjekter gjennom git for man full oversikt over prosjektets endring ved hjelp av versjonskontroll. Siden versjonskontroll er distribuert og ikke mappehieraki vil den ha en oversikt over filene og historikken til enhver tid. Ved evt feil er det mulighet for å tilbakestille disse. Det blir derfor lettere når flere samarbeider om samme kode samtidig som gjør at man vil spare tid i det lengere løp.

Konseptet med git er hvordan den tenker på data. Git ser på daten som et sett med bilder av mini filsystemer. Hver gang du commit-er, eller lagrer tilstanden for prosjektet ditt i gir, så tar den i hovedsak bilde av hvordan alle filene dine ser ut der og da. Og lager en referanse til det bilde. Git lagrer bare filen hvis den oppdager endringer på filen. Hvis du bare lager filen uten å endre henviser den bare til den siste filen den laget.

For å oppsummere hvorfor bruke Git og GitHub? Er man webdesigner kan det være smart å bruk git slik at man beholder alle tidligere versjoner om feil skulle oppstå. Ved å tilbakestille prosjektet til en tidligere versjon kan vi se hvem som sist endret og finnet ut hvor feilen ligger. Andre fordeler med Git og GitHub er:

  • Sparer tid i det lange løpet
  • Lettere for flere å samarbeide om samme kode
  • Muligheter til tilbakestilling ved evt feil
  • Henter informasjon om filer og proskjeter raskt og effektivt
